091/**
092 * A distributed {@link org.jsimpledb.kv.KVDatabase} based on the Raft consensus algorithm.
093 *
094 * <p>
095 * Raft defines a distributed consensus algorithm for maintaining a shared state machine.
096 * Each Raft node maintains a complete copy of the state machine. Cluster nodes elect a
097 * leader who manages updates to the state machine and provides for consistent readds.
098 * As long as as a node is part of a majority, the state machine is fully operational.
099 *
100 * <p>
101 * {@link RaftKVDatabase} turns
102 * this into a transactional key/value database with linearizable ACID semantics:
103 *  <ul>
104 *  <li>The Raft state machine is the key/value store data.</li>
105 *  <li>Unapplied log entries are stored on disk as serialized {@link Writes}, and also cached in memory.</li>
106 *  <li>On leaders only, the {@link Writes} associated with some number of the most recently applied log entries
107 *      are cached in memory for time <i>T<sub>max</sub></i> since creation, where <i>T<sub>max</sub></i> is the
108 *      maximum supported transaction duration (see below for how these cached {@link Writes} are used).
109 *      This caching is subject to <i>M<sub>max</sub></i>, which limits the total memory used by memory-cached
110 *      {@link Writes}; if reached, these cached {@link Writes} are discarded early.</li>
111 *  <li>Concurrent transactions are supported through a simple optimistic locking MVCC scheme (same as used by
112 *      {@link org.jsimpledb.kv.mvcc.SnapshotKVDatabase}):
113 *      <ul>
114 *      <li>Transactions execute locally until commit time, using a {@link MutableView} to collect mutations.
115 *          The {@link MutableView} is based on the local node's most recent log entry (whether committed or not);
116 *          this is called the <i>base term and index</i> for the transaction.</li>
117 *      <li>On commit, the transaction's {@link Reads}, {@link Writes}, and base index and term are
118 *          {@linkplain CommitRequest sent} to the leader.</li>
119 *      <li>The leader confirms that the log entry corresponding to the transaction's base index and term either:
120 *          <ul>
121 *          <li>Is still present in its own log and not yet applied to the state machine (in which case the
122 *              {@link Writes} are available with the log entry); or</li>
123 *          <li>Corresponds to an already applied log entry whose {@link Writes} are still cached in memory.</li>
124 *          </ul>
125 *          If this is not the case, then the transaction's base log entry is too old (older than <i>T<sub>max</sub></i>,
126 *          or was committed early due to memory pressure), and so the transaction is rejected with a
127 *          {@link RetryTransactionException}. Otherwise, the {@link Writes} associated with all log entries
128 *          after the transaction's base log entry are available.
129 *      <li>The leader confirms that the {@link Writes} associated with log entries created after the transaction's
130 *          base log entry do not create linearizability violations when {@linkplain Reads#isConflict compared against}
131 *          the transaction's {@link Reads}. If so, the transaction is rejected with a retry.</li>
132 *      <li>The leader adds a new log entry consisting of the transaction's {@link Writes} to its log.
133 *          The associated term and index become the transaction's <i>commit term and index</i>; the leader
134 *          then {@linkplain CommitResponse replies} to the follower with this information.</li>
135 *      <li>When the follower sees a log entry appear in its log matching the transaction's commit term and index, and
136 *          that log entry has been committed (in the Raft sense), the transaction is also committed.</li>
137 *      <li>As an optimization, when the leader sends a log entry to the same follower who committed the corresponding
138 *          transaction in the first place, only the transaction ID is sent, because the follower already has the data.</li>
139 *      </ul>
140 *  </li>
141 *  <li>For transactions occurring on a leader, the logic is the same except of course no network communication occurs.</li>
142 *  <li>For read-only transactions, the leader does not create a new log entry; instead, the transaction's commit
143 *      term and index are set to the term and index of the last entry in the leader's log. The leader also calculates its
144 *      current "leader lease timeout", which is the earliest time at which it is possible for another leader to be elected.
145 *      This is calculated as the time in the past at which the leader sent {@link AppendRequest}'s to a majority of followers
146 *      who have since responded, plus the {@linkplain #setMinElectionTimeout minimum election timeout}, minus a small adjustment
147 *      for possible clock drift (this assumes all nodes have the same minimum election timeout configured). If the current
148 *      time is prior to the leader lease timeout, the transaction may be committed immediately; otherwise, the current time
149 *      is returned to the follower as minimum required leader lease timeout before the transaction may be committed.</li>
150 *  <li>Included with each {@link AppendRequest} is the leader's current timestamp and lease timeout, so followers can commit
151 *      any waiting read-only transactions. Leaders keep track of which followers are waiting on which leader lease
152 *      timeout values, and when the leader lease timeout advances to allow a follower to commit a transaction, the follower
153 *      is notified with an immediate {@link AppendRequest} update.</li>
154 *  <li>A weaker consistency guarantee for read-only transactions (serializable but not linearizable) is also possible; see
155 *      {@link RaftKVTransaction#setReadOnlySnapshot RaftKVTransaction.setReadOnlySnapshot()}.</li>
156 *  </ul>
157 *
158 * <p><b>Limitations</b></p>
159 *
160 * <ul>
161 *  <li>A transaction's mutations must fit in memory.</li>
162 *  <li>An {@link AtomicKVStore} is required to store local persistent state;
163 *      if none is configured, a {@link LevelDBKVStore} is used.</li>
164 *  <li>All nodes must be configured with the same {@linkplain #setMinElectionTimeout minimum election timeout}.</li>
165 *  <li>Due to the optimistic locking approach used, this implementation will perform poorly when there is a high
166 *      rate of conflicting transactions; the result will be many transaction retries.</li>
167 *  <li>Performance will suffer when the amount of data associated with a typical transaction cannot be delivered
168 *      quickly and reliably over the network.</li>
169 * </ul>
170 *
171 * <p><b>Cluster Configuration</b></p>
172 *
173 * <p>
174 * Instances support dynamic cluster configuration changes at runtime. Initially, all nodes are in an <i>unconfigured</i>
175 * state, where nothing has been added to the Raft log yet and no cluster is defined. Unconfigured nodes are passive: they
176 * stay in follower mode (i.e., they will not start elections), and they disallow local transactions that make any changes
177 * (other than as described below to initialize a new cluster).
178 *
179 * <p>
180 * A node is configured if and only if it has a <i>configuration</i>. A cluster configuration consists of the identites of
181 * the nodes in the cluster, and their corresponding network addresses. Once a node <i>configured</i>, a separate issue
182 * is whether a node is <i>included</i> in its own configuration. A node that is not included in its own configuration
183 * does not count its own vote for elections and log commits, but otherwise functions normally in the cluster.
184 *
185 * <p>
186 * An unconfigured node becomes configured when either:
187 * <ul>
188 *  <li>An {@link AppendRequest} is received from a leader of some existing cluster, in which case the node
189 *      joins the cluster and applies the received cluster configuration; or</li>
190 *  <li>{@link RaftKVTransaction#configChange RaftKVTransaction.configChange()} is invoked and committed within
191 *      a local transaction, which creates a new cluster and configures it to contain the local node</li>
192 * </ul>
193 *
194 * <p>
195 * On the first commit, newly created clusters are assigned a random 32-bit cluster ID. This ID is included in all
196 * messages sent over the network, and adopted by unconfigured nodes that join the cluster. Configured nodes will discard
197 * incoming messages containing a different cluster ID. This prevents data corruption that can occur if nodes from two
198 * different clusters are inadvertently "mixed" together.
199 *
200 * <p>
201 * Once a node joins a cluster (identified by cluster ID), it cannot be reassigned to a different cluster without first
202 * returning it to the unconfigured state; to do that, it must be shut it down and its persistent state directory deleted.
203 *
204 * @see <a href="https://raftconsensus.github.io/">The Raft Consensus Algorithm</a>
205 */

514     * Start this instance.
515     *
516     * <p>
517     * Does nothing if already started or in the process of shutting down.
518     * </p>
519     *
520     * @throws IllegalStateException if this instance is not properly configured
521     * @throws IllegalArgumentException if persistent data is corrupted
522     * @throws IOException if an I/O error occurs
523     */
524    @PostConstruct
525    public synchronized void start() throws IOException {
526
527        // Sanity check
528        assert this.checkState();
529        if (this.role != null)
530            return;
531        Preconditions.checkState(!this.shuttingDown, "shutdown in progress");
532        Preconditions.checkState(this.logDir != null, "no log directory configured");
533        Preconditions.checkState(this.network != null, "no network configured");
534        Preconditions.checkState(this.kv == null, "key/value store exists");
535        Preconditions.checkState(this.minElectionTimeout <= this.maxElectionTimeout, "minElectionTimeout > maxElectionTimeout");
536        Preconditions.checkState(this.heartbeatTimeout < this.minElectionTimeout, "heartbeatTimeout >= minElectionTimeout");
537        Preconditions.checkState(this.identity != null, "no identity configured");
538
539        // Log
540        this.info("starting " + this + " in directory " + this.logDir);
541
542        // Start up local database
543        boolean success = false;
544        try {
545
546            // Create/verify log directory
547            if (!this.logDir.exists() && !this.logDir.isDirectory())
548                throw new IOException("failed to create directory `" + this.logDir + "'");
549            if (!this.logDir.isDirectory())
550                throw new IOException("file `" + this.logDir + "' is not a directory");
551
552            // By default use an atomic key/value store based on LevelDB if not explicitly specified
553            if (this.kvstore != null)
554                this.kv = this.kvstore;
555            else {
556                final File leveldbDir = new File(this.logDir, "levedb" + KVSTORE_FILE_SUFFIX);
557                if (!leveldbDir.exists() && !leveldbDir.mkdirs())
558                    throw new IOException("failed to create directory `" + leveldbDir + "'");
559                if (!leveldbDir.isDirectory())
560                    throw new IOException("file `" + leveldbDir + "' is not a directory");
561                this.kv = new LevelDBKVStore(new Iq80DBFactory().open(leveldbDir, new Options().createIfMissing(true)));
562            }
563
564            // Open directory containing log entry files so we have a way to fsync() it
565            assert this.logDirChannel == null;
566            this.logDirChannel = FileChannel.open(this.logDir.toPath(), StandardOpenOption.READ);
567
568            // Create randomizer
569            assert this.random == null;
570            this.random = new SecureRandom();
571
572            // Start up executor thread
573            assert this.executor == null;
574            this.executor = Executors.newSingleThreadScheduledExecutor(new ThreadFactory() {
575                @Override
576                public Thread newThread(Runnable action) {
577                    return new ServiceThread(action);
578                }
579            });
580
581            // Start network
582            this.network.start(new Network.Handler() {
583                @Override
584                public void handle(String sender, ByteBuffer buf) {
585                    RaftKVDatabase.this.handle(sender, buf);
586                }
587
588                @Override
589                public void outputQueueEmpty(String address) {
590                    RaftKVDatabase.this.outputQueueEmpty(address);
591                }
592            });
593
594            // Reload persistent raft info
595            this.clusterId = (int)this.decodeLong(CLUSTER_ID_KEY, 0);
596            this.currentTerm = this.decodeLong(CURRENT_TERM_KEY, 0);
597            final String votedFor = this.decodeString(VOTED_FOR_KEY, null);
598            this.lastAppliedTerm = this.decodeLong(LAST_APPLIED_TERM_KEY, 0);
599            this.lastAppliedIndex = this.decodeLong(LAST_APPLIED_INDEX_KEY, 0);
600            this.lastAppliedConfig = this.decodeConfig(LAST_APPLIED_CONFIG_KEY);
601            this.currentConfig = this.buildCurrentConfig();
602
603            // If we crashed part way through a snapshot install, recover by resetting state machine
604            if (this.lastAppliedTerm < 0 || this.lastAppliedIndex < 0) {
605                this.info("detected partially applied snapshot, resetting state machine");
606                if (!this.resetStateMachine(true))
607                    throw new IOException("error resetting state machine");
608            }
609
610            // Initialize commit index
611            this.commitIndex = this.lastAppliedIndex;
612
613            // Reload outstanding log entries from disk
614            this.loadLog();
615
616            // Show recovered state
617            this.info("recovered Raft state:"
618              + "\n  clusterId=" + (this.clusterId != 0 ? String.format("0x%08x", this.clusterId) : "none")
619              + "\n  currentTerm=" + this.currentTerm
620              + "\n  lastApplied=" + this.lastAppliedTerm + "/" + this.lastAppliedIndex
621              + "\n  lastAppliedConfig=" + this.lastAppliedConfig
622              + "\n  currentConfig=" + this.currentConfig
623              + "\n  votedFor=" + (votedFor != null ? "\"" + votedFor + "\"" : "nobody")
624              + "\n  log=" + this.raftLog);
625
626            // Validate recovered state
627            if (this.isConfigured()) {
628                Preconditions.checkArgument(this.clusterId != 0);
629                Preconditions.checkArgument(this.currentTerm > 0);
630                Preconditions.checkArgument(this.getLastLogTerm() > 0);
631                Preconditions.checkArgument(this.getLastLogIndex() > 0);
632                Preconditions.checkArgument(!this.currentConfig.isEmpty());
633            } else {
634                Preconditions.checkArgument(this.lastAppliedTerm == 0);
635                Preconditions.checkArgument(this.lastAppliedIndex == 0);
636                Preconditions.checkArgument(this.currentTerm == 0);
637                Preconditions.checkArgument(this.getLastLogTerm() == 0);
638                Preconditions.checkArgument(this.getLastLogIndex() == 0);
639                Preconditions.checkArgument(this.currentConfig.isEmpty());
640                Preconditions.checkArgument(this.raftLog.isEmpty());
641            }
642
643            // Start as follower (with unknown leader)
644            this.changeRole(new FollowerRole(this, null, null, votedFor));
645
646            // Done
647            success = true;
648        } finally {
649            if (!success)
650                this.cleanup();
651        }
652
653        // Sanity check
654        assert this.checkState();
655    }

657    /**
658     * Stop this instance.
659     *
660     * <p>
661     * Does nothing if not {@linkplain #start started} or in the process of shutting down.
662     * </p>
663     */
664    @PreDestroy
665    public void stop() {
666
667        // Set flag to prevent new transactions
668        synchronized (this) {
669
670            // Sanity check
671            assert this.checkState();
672            if (this.role == null || this.shuttingDown)
673                return;
674
675            // Set shutting down flag
676            this.info("starting shutdown of " + this);
677            this.shuttingDown = true;
678
679            // Fail all remaining open transactions
680            for (RaftKVTransaction tx : new ArrayList<RaftKVTransaction>(this.openTransactions.values())) {
681                switch (tx.getState()) {
682                case EXECUTING:
683                    tx.rollback();
684                    break;
685                case COMMIT_READY:
686                case COMMIT_WAITING:
687                    this.fail(tx, new KVTransactionException(tx, "database shutdown"));
688                    break;
689                case COMPLETED:
690                    break;
691                default:
692                    assert false;
693                    break;
694                }
695            }
696
697            // Sleep while we wait for transactions to clean themselves up
698            boolean done = false;
699            try {
700                done = TimedWait.wait(this, 5000, new org.dellroad.stuff.java.Predicate() {
701                    @Override
702                    public boolean test() {
703                        return RaftKVDatabase.this.openTransactions.isEmpty();
704                    }
705                });
706            } catch (InterruptedException e) {
707                Thread.currentThread().interrupt();
708            }
709            if (!done)
710                this.warn("open transactions not cleaned up during shutdown");
711        }
712
713        // Shut down the executor and wait for pending tasks to finish
714        this.executor.shutdownNow();
715        try {
716            this.executor.awaitTermination(1000, TimeUnit.MILLISECONDS);
717        } catch (InterruptedException e) {
718            Thread.currentThread().interrupt();
719        }
720
721        // Final cleanup
722        synchronized (this) {
723            this.executor = null;
724            this.cleanup();
725        }
726
727        // Done
728        this.info("completed shutdown of " + this);
729    }

731    private void cleanup() {
732        assert Thread.holdsLock(this);
733        assert this.openTransactions.isEmpty();
734        if (this.role != null) {
735            this.role.shutdown();
736            this.role = null;
737        }
738        if (this.executor != null) {
739            this.executor.shutdownNow();
740            try {
741                this.executor.awaitTermination(1000, TimeUnit.MILLISECONDS);
742            } catch (InterruptedException e) {
743                Thread.currentThread().interrupt();
744            }
745            this.executor = null;
746        }
747        if (this.kv != null) {
748            if (this.kvstore == null) {
749                final LevelDBKVStore ldb = (LevelDBKVStore)this.kv;
750                Util.closeIfPossible(ldb);
751                Util.closeIfPossible(ldb.getDB());
752            }
753            this.kv = null;
754        }
755        Util.closeIfPossible(this.logDirChannel);
756        this.logDirChannel = null;
757        this.raftLog.clear();
758        this.random = null;
759        this.network.stop();
760        this.currentTerm = 0;
761        this.commitIndex = 0;
762        this.clusterId = 0;
763        this.lastAppliedTerm = 0;
764        this.lastAppliedIndex = 0;
765        this.lastAppliedConfig = null;
766        this.currentConfig = null;
767        this.transmitting.clear();
768        this.pendingService.clear();
769        this.shuttingDown = false;
770    }

772    /**
773     * Initialize our in-memory state from the persistent state reloaded from disk.
774     * This is invoked on initial startup.
775     */
776    private void loadLog() throws IOException {
777
778        // Sanity check
779        assert Thread.holdsLock(this);
780
781        // Scan for log entry files
782        this.raftLog.clear();
783        for (File file : this.logDir.listFiles()) {
784
785            // Is this a log entry file?
786            if (LogEntry.LOG_FILE_PATTERN.matcher(file.getName()).matches()) {
787                this.info("recovering log file " + file.getName());
788                this.raftLog.add(LogEntry.fromFile(file));
789                continue;
790            }
791
792            // Is this a leftover temporary file?
793            if (TEMP_FILE_PATTERN.matcher(file.getName()).matches()) {
794                this.info("deleting leftover temporary file " + file.getName());
795                if (!file.delete())
796                    this.error("failed to delete leftover temporary file " + file);
797                continue;
798            }
799
800            // Is this a KV store directory (expected)?
801            if (KVSTORE_FILE_PATTERN.matcher(file.getName()).matches())
802                continue;
803
804            // Unknown
805            this.warn("ignoring unrecognized file " + file.getName() + " in my log directory");
806        }
807
808        // Verify we have a contiguous range of log entries starting from the snapshot index; discard bogus log files
809        Collections.sort(this.raftLog, LogEntry.SORT_BY_INDEX);
810        long lastTermSeen = this.lastAppliedTerm;
811        long expectedIndex = this.lastAppliedIndex + 1;
812        for (Iterator<LogEntry> i = this.raftLog.iterator(); i.hasNext(); ) {
813            final LogEntry logEntry = i.next();
814            String error = null;
815            if (logEntry.getTerm() < lastTermSeen)
816                error = "term " + logEntry.getTerm() + " < last applied term " + lastTermSeen;
817            else if (logEntry.getIndex() < this.lastAppliedIndex)
818                error = "index " + logEntry.getIndex() + " < last applied index " + this.lastAppliedIndex;
819            else if (logEntry.getIndex() != expectedIndex)
820                error = "index " + logEntry.getIndex() + " != expected index " + expectedIndex;
821            if (error != null) {
822                this.warn("deleting bogus log file " + logEntry.getFile().getName() + ": " + error);
823                if (!logEntry.getFile().delete())
824                    this.error("failed to delete bogus log file " + logEntry.getFile());
825                i.remove();
826            } else {
827                expectedIndex++;
828                lastTermSeen = logEntry.getTerm();
829            }
830        }
831        this.info("recovered " + this.raftLog.size() + " Raft log entries: " + this.raftLog);
832
833        // Rebuild current configuration
834        this.currentConfig = this.buildCurrentConfig();
835    }

918     * @throws IllegalStateException if this instance is not {@linkplain #start started} or in the process of shutting down

920    @Override
921    public synchronized RaftKVTransaction createTransaction() {
922
923        // Sanity check
924        assert this.checkState();
925        Preconditions.checkState(this.role != null, "not started");
926        Preconditions.checkState(!this.shuttingDown, "shutting down");
927
928        // Base transaction on the most recent log entry. This is itself a form of optimistic locking: we assume that the
929        // most recent log entry has a high probability of being committed (in the Raft sense), which is of course required
930        // in order to commit any transaction based on it.
931        final MostRecentView view = new MostRecentView();
932
933        // Create transaction
934        final RaftKVTransaction tx = new RaftKVTransaction(this,
935          this.getLastLogTerm(), this.getLastLogIndex(), view.getSnapshot(), view.getView());
936        if (this.log.isDebugEnabled())
937            this.debug("created new transaction " + tx);
938        this.openTransactions.put(tx.getTxId(), tx);
939
940        // Set default commit timeout
941        tx.setTimeout(this.commitTimeout);
942
943        // Done
944        return tx;
945    }

947    /**
948     * Commit a transaction.
949     */
950    void commit(final RaftKVTransaction tx) {
951        try {
952
953            // Mark transaction as "commit ready" - service thread will do the rest
954            synchronized (this) {
955
956                // Sanity check
957                assert this.checkState();
958                assert this.role != null;
959
960                // Check tx state
961                switch (tx.getState()) {
962                case EXECUTING:
963
964                    // Transition to COMMIT_READY state
965                    if (this.log.isDebugEnabled())
966                        this.debug("committing transaction " + tx);
967                    tx.closeSnapshot();
968                    tx.setState(TxState.COMMIT_READY);
969                    this.requestService(this.role.new CheckReadyTransactionService(tx));
970
971                    // Setup commit timer
972                    final int timeout = tx.getTimeout();
973                    if (timeout != 0) {
974                        final Timer commitTimer = new Timer("commit timer for " + tx,
975                          new Service(null, "commit timeout for tx#" + tx.getTxId()) {
976                            @Override
977                            public void run() {
978                                switch (tx.getState()) {
979                                case COMMIT_READY:
980                                case COMMIT_WAITING:
981                                    RaftKVDatabase.this.fail(tx, new RetryTransactionException(tx,
982                                      "transaction failed to complete within " + timeout + "ms (in state " + tx.getState() + ")"));
983                                    break;
984                                default:
985                                    break;
986                                }
987                            }
988                        });
989                        commitTimer.timeoutAfter(tx.getTimeout());
990                        tx.setCommitTimer(commitTimer);
991                    }
992                    break;
993                case CLOSED:                                        // this transaction has already been committed or rolled back
994                    throw new StaleTransactionException(tx);
995                default:                                            // another thread is already doing the commit
996                    this.warn("simultaneous commit()'s requested for " + tx + " by two different threads");
997                    break;
998                }
999            }
1000
1001            // Wait for completion
1002            try {
1003                tx.getCommitFuture().get();
1004            } catch (InterruptedException e) {
1005                throw new RetryTransactionException(tx, "thread interrupted while waiting for commit", e);
1006            } catch (ExecutionException e) {
1007                final Throwable cause = e.getCause();
1008                Util.prependCurrentStackTrace(cause, "Asynchronous Commit");
1009                if (cause instanceof Error)
1010                    throw (Error)cause;
1011                if (cause instanceof RuntimeException)
1012                    throw (RuntimeException)cause;
1013                throw new KVTransactionException(tx, "commit failed", cause);           // should never get here
1014            }
1015        } finally {
1016            this.cleanupTransaction(tx);
1017        }
1018    }

1021     * Rollback a transaction.
1022     */
1023    synchronized void rollback(RaftKVTransaction tx) {
1024
1025        // Sanity check
1026        assert this.checkState();
1027        assert this.role != null;
1028
1029        // Check tx state
1030        switch (tx.getState()) {
1031        case EXECUTING:
1032            if (this.log.isDebugEnabled())
1033                this.debug("rolling back transaction " + tx);
1034            this.cleanupTransaction(tx);
1035            break;
1036        case CLOSED:
1037            break;
1038        default:                                            // another thread is currently committing!
1039            this.warn("simultaneous commit() and rollback() requested for " + tx + " by two different threads");
1040            break;
1041        }
1042    }

1044    private synchronized void cleanupTransaction(RaftKVTransaction tx) {
1045
1046        // Debug
1047        if (this.log.isTraceEnabled())
1048            this.trace("cleaning up transaction " + tx);
1049
1050        // Do any per-role cleanups
1051        if (this.role != null)
1052            this.role.cleanupForTransaction(tx);
1053
1054        // Close transaction's snapshot
1055        tx.closeSnapshot();
1056
1057        // Cancel commit timer
1058        final Timer commitTimer = tx.getCommitTimer();
1059        if (commitTimer != null)
1060            commitTimer.cancel();
1061
1062        // Remove from open transactions set
1063        this.openTransactions.remove(tx.getTxId());
1064
1065        // Transition to CLOSED
1066        tx.setState(TxState.CLOSED);
1067
1068        // Notify waiting thread if doing shutdown
1069        if (this.shuttingDown)
1070            this.notify();
1071    }

1075

1110     * A view of the database based on the most recent log entry, if any, otherwise directly on the committed key/value store.
1111     * Caller is responsible for eventually closing the snapshot.
1112     */
1113    private class MostRecentView {
1114
1115        private final CloseableKVStore snapshot;
1116        private final MutableView view;
1117
1118        public MostRecentView() {
1119            assert Thread.holdsLock(RaftKVDatabase.this);
1120
1121            // Grab a snapshot of the key/value store
1122            this.snapshot = RaftKVDatabase.this.kv.snapshot();
1123
1124            // Create a view of just the state machine keys and values and successively layer unapplied log entries
1125            KVStore kview = PrefixKVStore.create(snapshot, STATE_MACHINE_PREFIX);
1126            for (LogEntry logEntry : RaftKVDatabase.this.raftLog) {
1127                final Writes writes = logEntry.getWrites();
1128                if (!writes.isEmpty())
1129                    kview = new MutableView(kview, null, logEntry.getWrites());
1130            }
1131
1132            // Finalize
1133            this.view = new MutableView(kview);
1134        }
1135
1136        public CloseableKVStore getSnapshot() {
1137            return this.snapshot;
1138        }
1139
1140        public MutableView getView() {
1141            return this.view;
1142        }
1143    }

1147    /**
1148     * Request service to be invoked after the current service (if any) completes.
1149     *
1150     * <p>
1151     * If {@code service} has an associated {@link Role}, and that {@link Role} is no longer active
1152     * when the service is handled, nothing will be done.
1153     *
1154     * @param service the service to perform
1155     */
1156    private void requestService(Service service) {
1157        assert Thread.holdsLock(this);
1158        if (!this.pendingService.add(service))
1159            return;
1160        if (this.performingService)
1161            return;
1162        try {
1163            this.executor.submit(new ErrorLoggingRunnable() {
1164                @Override
1165                protected void doRun() {
1166                    RaftKVDatabase.this.handlePendingService();
1167                }
1168            });
1169        } catch (RejectedExecutionException e) {
1170            if (!this.shuttingDown)
1171                this.warn("executor task rejected, skipping", e);
1172        }
1173    }

1175    // Performs pending service requests (do not invoke directly)
1176    private synchronized void handlePendingService() {
1177
1178        // Sanity check
1179        assert this.checkState();
1180        assert Thread.currentThread() instanceof ServiceThread;
1181        if (this.role == null)
1182            return;
1183
1184        // While there is work to do, do it
1185        this.performingService = true;
1186        try {
1187            while (!this.pendingService.isEmpty()) {
1188                final Iterator<Service> i = this.pendingService.iterator();
1189                final Service service = i.next();
1190                i.remove();
1191                assert service.getRole() == null || service.getRole() == this.role;
1192                if (this.log.isTraceEnabled())
1193                    this.trace("SERVICE [" + service + "] in " + this.role);
1194                service.run();
1195            }
1196        } finally {
1197            this.performingService = false;
1198        }
1199    }

1230    /**
1231     * One shot timer that {@linkplain #requestService requests} a {@link Service} on expiration.
1232     */
1233    class Timer {
1234
1235        private final Logger log = RaftKVDatabase.this.log;
1236        private final String name;
1237        private final Service service;
1238        private ScheduledFuture<?> future;
1239        private PendingTimeout pendingTimeout;                  // non-null IFF timeout has not been handled yet
1240        private Timestamp timeoutDeadline;
1241
1242        public Timer(String name, Service service) {
1243            this.name = name;
1244            this.service = service;
1245        }
1246
1247        /**
1248         * Stop timer if running.
1249         *
1250         * @throws IllegalStateException if the lock object is not locked
1251         */
1252        public void cancel() {
1253
1254            // Sanity check
1255            assert Thread.holdsLock(RaftKVDatabase.this);
1256
1257            // Cancel existing timer, if any
1258            if (this.future != null) {
1259                this.future.cancel(false);
1260                this.future = null;
1261            }
1262
1263            // Ensure the previously scheduled action does nothing if case we lose the cancel() race condition
1264            this.pendingTimeout = null;
1265            this.timeoutDeadline = null;

1270         *

1272         * @return true if restarted, false if executor rejected the task
1273         * @throws IllegalStateException if the lock object is not locked
1274         */
1275        public void timeoutAfter(int delay) {
1276

1278            assert Thread.holdsLock(RaftKVDatabase.this);
1279            Preconditions.checkArgument(delay >= 0, "delay < 0");
1280

1282            this.cancel();

1284            assert this.pendingTimeout == null;
1285            assert this.timeoutDeadline == null;

1288            this.timeoutDeadline = new Timestamp().offset(delay);

1290                RaftKVDatabase.this.trace("rescheduling " + this.name + " for " + this.timeoutDeadline

1292            }

1295                this.future = RaftKVDatabase.this.executor.schedule(this.pendingTimeout, delay, TimeUnit.MILLISECONDS);
1296            } catch (RejectedExecutionException e) {
1297                if (!RaftKVDatabase.this.shuttingDown)
1298                    RaftKVDatabase.this.warn("can't restart timer", e);
1299            }

1330                RaftKVDatabase.this.trace(Timer.this.name + " expired " + -this.timeoutDeadline.offsetFromNow() + "ms ago");

1726

1728        assert Thread.holdsLock(this);
1729

1753    private synchronized void receiveMessage(String address, Message msg) {
1754
1755        // Sanity check
1756        assert Thread.holdsLock(this);
1757        assert this.checkState();
1758        if (this.role == null) {
1759            if (this.log.isDebugEnabled())
1760                this.debug("rec'd " + msg + " rec'd in shutdown state; ignoring");
1761            return;
1762        }
1763
1764        // Sanity check cluster ID
1765        if (msg.getClusterId() == 0) {
1766            this.warn("rec'd " + msg + " with zero cluster ID from " + address + "; ignoring");
1767            return;
1768        }
1769        if (this.clusterId != 0 && msg.getClusterId() != this.clusterId) {
1770            this.warn("rec'd " + msg + " with foreign cluster ID "
1771              + String.format("0x%08x", msg.getClusterId()) + " != " + String.format("0x%08x", this.clusterId) + "; ignoring");
1772            return;
1773        }
1774
1775        // Sanity check sender
1776        final String peer = msg.getSenderId();
1777        if (peer.equals(this.identity)) {
1778            this.warn("rec'd " + msg + " from myself (\"" + peer + "\", address " + address + "); ignoring");
1779            return;
1780        }
1781
1782        // Sanity check recipient
1783        final String dest = msg.getRecipientId();
1784        if (!dest.equals(this.identity)) {
1785            this.warn("rec'd misdirected " + msg + " intended for \"" + dest + "\" from " + address + "; ignoring");
1786            return;
1787        }
1788
1789        // Is sender's term too low? Ignore it
1790        if (msg.getTerm() < this.currentTerm) {
1791            this.info("rec'd " + msg + " with term " + msg.getTerm() + " < " + this.currentTerm + " from \""
1792              + peer + "\" at " + address + ", ignoring");
1793            return;
1794        }
1795
1796        // Is my term too low? If so update and revert to follower
1797        if (msg.getTerm() > this.currentTerm) {
1798
1799            // First check with current role; in some special cases we ignore this
1800            if (!this.role.mayAdvanceCurrentTerm(msg)) {
1801                if (this.log.isTraceEnabled()) {
1802                    this.trace("rec'd " + msg + " with term " + msg.getTerm() + " > " + this.currentTerm + " from \""
1803                      + peer + "\" but current role says to ignore it");
1804                }
1805                return;
1806            }
1807
1808            // Revert to follower
1809            this.info("rec'd " + msg.getClass().getSimpleName() + " with term " + msg.getTerm() + " > " + this.currentTerm
1810              + " from \"" + peer + "\", updating term and "
1811              + (this.role instanceof FollowerRole ? "remaining a" : "reverting to") + " follower");
1812            if (!this.advanceTerm(msg.getTerm()))
1813                return;
1814            this.changeRole(msg.isLeaderMessage() ? new FollowerRole(this, peer, address) : new FollowerRole(this));
1815        }
1816
1817        // Debug
1818        if (this.log.isTraceEnabled())
1819            this.trace("RECV " + msg + " in " + this.role + " from " + address);
1820
1821        // Handle message
1822        this.returnAddress = address;
1823        try {
1824            msg.visit(this.role);
1825        } finally {
1826            this.returnAddress = null;
1827        }
1828    }

2190        /**
2191         * Check a transaction waiting for its log entry to be committed (in the {@link TxState#COMMIT_WAITING} state).
2192         *
2193         * <p>
2194         * This should be invoked:
2195         * <ul>
2196         *  <li>After changing roles</li>
2197         *  <li>After a transaction has entered the {@link TxState#COMMIT_WAITING} state</li>
2198         *  <li>After advancing my {@code commitIndex} (as leader or follower)</li>
2199         *  <li>After receiving an updated {@linkplain AppendResponse#getLeaderLeaseTimeout leader lease timeout}
2200         *      (in {@link FollowerRole})</li>
2201         * </ul>
2202         *
2203         * @param tx the transaction
2204         * @throws KVTransactionException if an error occurs
2205         */
2206        private void checkWaitingTransaction(RaftKVTransaction tx) {
2207
2208            // Handle the case the transaction's committed log index has already been applied to the state machine
2209            final long commitIndex = tx.getCommitIndex();
2210            if (commitIndex < this.raft.lastAppliedIndex) {
2211
2212                // This can happen if we lose contact and by the time we're back the log entry has
2213                // already been applied to the state machine on some leader and that leader sent
2214                // use an InstallSnapshot message. We don't know whether it actually got committed
2215                // or not, so the transaction must be retried.
2216                throw new RetryTransactionException(tx, "committed log entry was missed");
2217            }
2218
2219            // Has the transaction's log entry been received yet?
2220            if (commitIndex > this.raft.getLastLogIndex())
2221                return;
2222
2223            // Has the transaction's log entry been committed yet?
2224            if (commitIndex > this.raft.commitIndex)
2225                return;
2226
2227            // Verify the term of the committed log entry; if not what we expect, the log entry was overwritten by a new leader
2228            final long commitTerm = tx.getCommitTerm();
2229            if (this.raft.getLogTermAtIndex(commitIndex) != commitTerm)
2230                throw new RetryTransactionException(tx, "leader was deposed during commit and transaction's log entry overwritten");
2231
2232            // Check with subclass
2233            if (!this.mayCommit(tx))
2234                return;
2235
2236            // Transaction is officially committed now
2237            if (this.log.isTraceEnabled())
2238                this.trace("commit successful for " + tx + " (commit index " + this.raft.commitIndex + " >= " + commitIndex + ")");
2239            this.raft.succeed(tx);
2240        }

2536         *

2539         * <ul>
2540         *  <li>After any log entry has been added to the log, if we have zero followers</li>
2541         *  <li>After a log entry that contains a configuration change has been added to the log</li>
2542         *  <li>After a follower's {@linkplain Follower#getMatchIndex match index} has advanced</li>
2543         * </ul>
2544         */
2545        private void updateLeaderCommitIndex() {
2546
2547            // Update my commit index based on when a majority of cluster members have ack'd log entries
2548            final long lastLogIndex = this.raft.getLastLogIndex();
2549            while (this.raft.commitIndex < lastLogIndex) {
2550
2551                // Get the index in question
2552                final long index = this.raft.commitIndex + 1;
2553
2554                // Count the number of nodes which have a copy of the log entry at index
2555                int numVotes = this.raft.isClusterMember() ? 1 : 0;                         // count myself, if member
2556                for (Follower follower : this.followerMap.values()) {                       // count followers who are members
2557                    if (follower.getMatchIndex() >= index && this.raft.isClusterMember(follower.getIdentity()))
2558                        numVotes++;
2559                }
2560
2561                // Do a majority of cluster nodes have a copy of this log entry?
2562                final int majority = this.raft.currentConfig.size() / 2 + 1;
2563                if (numVotes < majority)
2564                    break;
2565
2566                // Log entry term must match my current term
2567                final LogEntry logEntry = this.raft.getLogEntryAtIndex(index);
2568                if (logEntry.getTerm() != this.raft.currentTerm)
2569                    break;
2570
2571                // Update commit index

2573                    this.debug("advancing commit index from " + this.raft.commitIndex + " -> " + index

2595        /**
2596         * Update my {@code leaseTimeout} based on followers' returned {@code leaderTimeout}'s.
2597         *
2598         * <p>
2599         * This should be invoked:
2600         * <ul>
2601         *  <li>After a follower has replied with an {@link AppendResponse} containing a newer
2602         *      {@linkplain AppendResponse#getLeaderTimestamp leader timestamp} than before</li>
2603         * </ul>
2604         */
2605        private void updateLeaseTimeout() {
2606
2607            // Only needed when we have followers
2608            final int numFollowers = this.followerMap.size();
2609            if (numFollowers == 0)
2610                return;
2611
2612            // Get all cluster member leader timestamps, sorted in increasing order
2613            final Timestamp[] leaderTimestamps = new Timestamp[this.raft.currentConfig.size()];
2614            int index = 0;
2615            if (this.raft.isClusterMember())
2616                leaderTimestamps[index++] = new Timestamp();                                // only matters in single node cluster
2617            for (Follower follower : this.followerMap.values()) {
2618                if (this.raft.isClusterMember(follower.getIdentity()))
2619                    leaderTimestamps[index++] = follower.getLeaderTimestamp();
2620            }
2621            Preconditions.checkArgument(index == leaderTimestamps.length);                  // sanity check
2622            Arrays.sort(leaderTimestamps);
2623
2624            //
2625            // Calculate highest leaderTimeout shared by a majority of cluster members, based on sorted array:
2626            //
2627            //  # nodes    timestamps
2628            //  -------    ----------
2629            //     5       [ ][ ][x][x][x]        3/5 x's make a majority at index (5 - 1)/2 = 2
2630            //     6       [ ][ ][x][x][x][x]     4/6 x's make a majority at index (6 - 1)/2 = 2
2631            //
2632            // The minimum leaderTimeout shared by a majority of nodes is at index (leaderTimestamps.length - 1) / 2.
2633            // We then add the minimum election timeout, then subtract a little for clock drift.
2634            //
2635            final Timestamp newLeaseTimeout = leaderTimestamps[(leaderTimestamps.length + 1) / 2]
2636              .offset((int)(this.raft.minElectionTimeout * (1.0f - MAX_CLOCK_DRIFT)));
2637
2638            // Immediately notify any followers who are waiting on this updated timestamp (or any smaller timestamp)
2639            if (newLeaseTimeout.compareTo(this.leaseTimeout) > 0) {
2640
2641                // Update my leader lease timeout
2642                if (this.log.isTraceEnabled())
2643                    this.trace("updating my lease timeout from " + this.leaseTimeout + " -> " + newLeaseTimeout);
2644                this.leaseTimeout = newLeaseTimeout;
2645
2646                // Notify any followers who care
2647                for (Follower follower : this.followerMap.values()) {
2648                    final NavigableSet<Timestamp> timeouts = follower.getCommitLeaseTimeouts().headSet(this.leaseTimeout, true);
2649                    if (!timeouts.isEmpty()) {
2650                        follower.updateNow();                           // notify follower so it can commit waiting transaction(s)
2651                        timeouts.clear();
2652                    }
2653                }
2654            }
2655        }

2659         *
2660         * <p>
2661         * This should be invoked:

2717        /**
2718         * Check whether a follower needs an update and send one if so.
2719         *
2720         * <p>
2721         * This should be invoked:
2722         * <ul>
2723         *  <li>After a new follower has been added</li>
2724         *  <li>When the output queue for a follower goes from non-empty to empty</li>
2725         *  <li>After the follower's {@linkplain Follower#getUpdateTimer update timer} has expired</li>
2726         *  <li>After a new log entry has been added to the log (all followers)</li>
2727         *  <li>After receiving an {@link AppendResponse} that caused the follower's
2728         *      {@linkplain Follower#getNextIndex next index} to change</li>
2729         *  <li>After receiving the first positive {@link AppendResponse} to a probe</li>
2730         *  <li>After our {@code commitIndex} has advanced (all followers)</li>
2731         *  <li>After our {@code leaseTimeout} has advanced past one or more of a follower's
2732         *      {@linkplain Follower#getCommitLeaseTimeouts commit lease timeouts} (with update timer reset)</li>
2733         *  <li>After sending a {@link CommitResponse} with a non-null {@linkplain CommitResponse#getCommitLeaderLeaseTimeout
2734         *      commit leader lease timeout} (all followers) to probe for updated leader timestamps</li>
2735         *  <li>After starting, aborting, or completing a snapshot install for a follower</li>
2736         * </ul>
2737         */
2738        private void updateFollower(Follower follower) {
2739
2740            // If follower has an in-progress snapshot that has become too stale, abort it
2741            final String peer = follower.getIdentity();
2742            SnapshotTransmit snapshotTransmit = follower.getSnapshotTransmit();
2743            if (snapshotTransmit != null && snapshotTransmit.getSnapshotIndex() < this.raft.lastAppliedIndex) {
2744                if (this.log.isDebugEnabled())
2745                    this.debug("aborting stale snapshot install for " + follower);
2746                follower.cancelSnapshotTransmit();
2747                follower.updateNow();
2748            }
2749
2750            // Is follower's queue empty? If not, hold off until then
2751            if (this.raft.isTransmitting(follower.getAddress())) {
2752                if (this.log.isTraceEnabled())
2753                    this.trace("no update for \"" + peer + "\": output queue still not empty");
2754                return;
2755            }
2756
2757            // Handle any in-progress snapshot install
2758            if ((snapshotTransmit = follower.getSnapshotTransmit()) != null) {
2759
2760                // Send the next chunk in transmission, if any
2761                final long pairIndex = snapshotTransmit.getPairIndex();
2762                final ByteBuffer chunk = snapshotTransmit.getNextChunk();
2763                boolean synced = true;
2764                if (chunk != null) {
2765
2766                    // Send next chunk
2767                    final InstallSnapshot msg = new InstallSnapshot(this.raft.clusterId, this.raft.identity, peer,
2768                      this.raft.currentTerm, snapshotTransmit.getSnapshotTerm(), snapshotTransmit.getSnapshotIndex(), pairIndex,
2769                      pairIndex == 0 ? snapshotTransmit.getSnapshotConfig() : null, !snapshotTransmit.hasMoreChunks(), chunk);
2770                    if (this.raft.sendMessage(msg))
2771                        return;
2772                    if (this.log.isDebugEnabled())
2773                        this.debug("canceling snapshot install for " + follower + " due to failure to send " + msg);
2774
2775                    // Message failed -> snapshot is fatally wounded, so cancel it
2776                    synced = false;
2777                }
2778                if (synced)
2779                    this.info("completed snapshot install for out-of-date " + follower);
2780
2781                // Snapshot transmit is complete (or failed)
2782                follower.cancelSnapshotTransmit();
2783
2784                // Trigger an immediate regular update
2785                follower.setNextIndex(snapshotTransmit.getSnapshotIndex() + 1);
2786                follower.setSynced(synced);
2787                follower.updateNow();
2788                this.raft.requestService(new UpdateFollowerService(follower));
2789                return;
2790            }
2791
2792            // Are we still waiting for the update timer to expire?
2793            if (!follower.getUpdateTimer().pollForTimeout()) {
2794                boolean waitForTimerToExpire = true;
2795
2796                // Don't wait for the update timer to expire if:
2797                //  (a) The follower is sync'd; AND
2798                //      (y) We have a new log entry that the follower doesn't have; OR
2799                //      (y) We have a new leaderCommit that the follower doesn't have
2800                // The effect is that we will pipeline updates to synchronized followers.
2801                if (follower.isSynced()
2802                  && (follower.getLeaderCommit() != this.raft.commitIndex
2803                   || follower.getNextIndex() <= this.raft.getLastLogIndex()))
2804                    waitForTimerToExpire = false;
2805
2806                // Wait for timer to expire
2807                if (waitForTimerToExpire) {
2808                    if (this.log.isTraceEnabled()) {
2809                        this.trace("no update for \"" + follower.getIdentity() + "\": timer not expired yet, and follower is "
2810                          + (follower.isSynced() ? "up to date" : "not synced"));
2811                    }
2812                    return;
2813                }
2814            }
2815
2816            // Get index of the next log entry to send to follower
2817            final long nextIndex = follower.getNextIndex();
2818
2819            // If follower is too far behind, we must do a snapshot install
2820            if (nextIndex <= this.raft.lastAppliedIndex) {
2821                final MostRecentView view = this.raft.new MostRecentView();
2822                follower.setSnapshotTransmit(new SnapshotTransmit(this.raft.getLastLogTerm(), this.raft.getLastLogIndex(),
2823                  this.raft.lastAppliedConfig, view.getSnapshot(), view.getView()));
2824                this.info("started snapshot install for out-of-date " + follower);
2825                this.raft.requestService(new UpdateFollowerService(follower));
2826                return;
2827            }
2828
2829            // Restart update timer here (to avoid looping if an error occurs below)
2830            follower.getUpdateTimer().timeoutAfter(this.raft.heartbeatTimeout);
2831
2832            // Send actual data if follower is synced and there is a log entry to send; otherwise, just send a probe
2833            final AppendRequest msg;
2834            if (!follower.isSynced() || nextIndex > this.raft.getLastLogIndex()) {
2835                msg = new AppendRequest(this.raft.clusterId, this.raft.identity, peer, this.raft.currentTerm, new Timestamp(),
2836                  this.leaseTimeout, this.raft.commitIndex, this.raft.getLogTermAtIndex(nextIndex - 1), nextIndex - 1);   // probe
2837            } else {
2838
2839                // Get log entry to send
2840                final LogEntry logEntry = this.raft.getLogEntryAtIndex(nextIndex);
2841
2842                // If the log entry correspond's to follower's transaction, don't send the data because follower already has it.
2843                // But only do this optimization the first time, in case something goes wrong on the follower's end.
2844                ByteBuffer mutationData = null;
2845                if (!follower.getSkipDataLogEntries().remove(logEntry)) {
2846                    try {
2847                        mutationData = logEntry.getContent();
2848                    } catch (IOException e) {
2849                        this.error("error reading log file " + logEntry.getFile(), e);
2850                        return;
2851                    }
2852                }
2853
2854                // Create message
2855                msg = new AppendRequest(this.raft.clusterId, this.raft.identity, peer, this.raft.currentTerm, new Timestamp(),
2856                  this.leaseTimeout, this.raft.commitIndex, this.raft.getLogTermAtIndex(nextIndex - 1), nextIndex - 1,
2857                  logEntry.getTerm(), mutationData);
2858            }
2859
2860            // Send update
2861            final boolean sent = this.raft.sendMessage(msg);
2862
2863            // Advance next index if a log entry was sent; we allow pipelining log entries when synchronized
2864            if (sent && !msg.isProbe()) {
2865                assert follower.isSynced();
2866                follower.setNextIndex(Math.min(follower.getNextIndex(), this.raft.getLastLogIndex()) + 1);
2867            }
2868
2869            // Update leaderCommit for follower
2870            if (sent)
2871                follower.setLeaderCommit(msg.getLeaderCommit());
2872        }
